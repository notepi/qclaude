# 商业航天情报系统 MVP 方案

## Goal

每日自动复盘铂力特（688393.SH），判断其在商业航天板块中的：
- 相对强弱（vs 板块整体）
- 资金关注度（主力动向、成交活跃度）

---

## MVP Scope

| 维度 | 范围 |
|------|------|
| 覆盖标的 | 仅铂力特 1 只股票 |
| 对比参照 | 商业航天板块（固定成分股列表） |
| 分析频率 | 每日一次（收盘后） |
| 输出形式 | Markdown 报告 |
| 数据来源 | 单一免费数据源（如东方财富/同花顺/AKShare） |

**一句话定位**：一个每天早上告诉你"铂力特昨天表现如何、资金怎么看"的自动化复盘工具。

---

## Modules（最小必要模块）

```
┌─────────────────────────────────────────────────────────┐
│                    space-intel-system                   │
├─────────────┬─────────────┬─────────────┬──────────────┤
│   fetcher   │  analyzer   │  reporter   │   storage    │
│  数据采集    │   分析计算   │   报告生成   │   数据存储    │
└─────────────┴─────────────┴─────────────┴──────────────┘
```

### Module 1: fetcher（数据采集）
- **职责**：拉取每日行情数据
- **输入**：股票代码列表
- **输出**：当日 OHLCV + 成交额 + 涨跌幅
- **频率**：每日 1 次

### Module 2: analyzer（分析计算）
- **职责**：计算核心指标
- **输入**：原始行情数据
- **输出**：指标结果（相对强弱分、资金关注度分等）

### Module 3: reporter（报告生成）
- **职责**：生成人类可读的复盘报告
- **输入**：分析结果
- **输出**：Markdown 报告文件

### Module 4: storage（数据存储）
- **职责**：持久化历史数据
- **形式**：SQLite 或 CSV 文件（MVP 阶段足够）
- **用途**：支持后续趋势分析

## Core Indicators（第一版核心指标）

### 1. 相对强弱指标

| 指标 | 计算方式 | 意义 |
|------|----------|------|
| 涨跌幅差 | 铂力特涨跌幅 - 板块平均涨跌幅 | 正值=强于板块，负值=弱于板块 |
| 排名位置 | 铂力特涨跌幅在板块成分股中的排名/总数 | 前 1/3 为强势 |

### 2. 资金关注度指标

| 指标 | 计算方式 | 意义 |
|------|----------|------|
| 成交额变化率 | 今日成交额 / 5日均值 - 1 | 放量或缩量 |
| 换手率 | 成交量 / 流通股本 | 交易活跃度 |
| 主力净流入 | 大单净买入额（如有） | 资金态度 |

### 3. 综合评分（可选简化版）

```
强弱分 = 涨跌幅差标准化(40%) + 排名分(30%) + 成交额变化率标准化(30%)
```

## Not In Scope（暂不做，留到后续）

| 功能 | 为什么不做 | 什么时候做 |
|------|-----------|-----------|
| 多股票监控 | MVP 只验证流程，先做好一只 | 当复盘流程稳定后 |
| 实时数据推送 | 不需要日内盯盘 | 当有日内交易需求时 |
| 复杂技术指标 | MACD/KDJ 等先不计算 | 先验证基础指标有用 |
| 舆情/新闻抓取 | 数据源复杂，MVP 不需 | 当基本面分析需要时 |
| 炒股软件对接 | 不做自动交易 | 长期规划 |
| Web 界面 | 先用 CLI + 文件输出 | 当有多用户需求时 |
| 回测系统 | 先不做策略验证 | 当需要评估指标有效性时 |
| 数据库优化 | CSV/SQLite 够用 | 当数据量大时 |
| 多数据源 | 先接一个稳定源 | 当单源不可靠时 |

## MVP 成功标准

- [ ] 每日自动生成一份复盘报告
- [ ] 报告包含：铂力特 vs 板块的相对强弱判断
- [ ] 报告包含：资金关注度变化的判断
- [ ] 历史数据可追溯（存储正常工作）
- [ ] 整个流程可一键触发

---

## 建议的实现顺序

1. **Day 1**: 搭建项目骨架 + 数据采集模块（能用 AKShare 拉数据）
2. **Day 2**: 实现指标计算 + 存储模块
3. **Day 3**: 实现报告生成 + 端到端测试
4. **Day 4**: 增加定时任务 + 异常处理
